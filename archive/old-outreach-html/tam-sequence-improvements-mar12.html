The problem

Evidence: sop-tam-outbound.md Part 5 time estimates + CLAUDE.md target = mathematical inconsistency. Actual observed send volume: peak day was 84 sends on Mar 12 (a multi-wave catch-up day), average is closer to 20-30 for a normal batch run.

2. Add to Part 5 of sop-tam-outbound.md: "Daily capacity note: With full Layer 1+2 research, realistic throughput is 3-4 accounts (9-12 contacts) per 2-hour session. Pre-built batch trackers (started prior session) expand this to 5-6 accounts."

3. Separate "prospecting day" (research + batch build) from "send day" (enroll + send from pre-built batch) in the SOP framing. These are different workflows.

14
No breakup email (Day 35) template exists — it's in the sequence but undocumented
sop-tam-outbound.md (missing section) apollo-config.md
The problem

TAM Outbound Step 7 (Day 35) is a breakup email. Wave 1 launched Feb 28, so breakup emails start hitting around April 4. There is no documented formula, example, or rule set for the breakup. The sequence task will appear in Apollo with no SOP guidance for what to write.

Evidence: apollo-config.md TAM Outbound sequence — Step 7 documented as "Breakup Email" with task note placeholder. sop-tam-outbound.md — no Part 9 or breakup section exists.
The fix
Add Part 9 to sop-tam-outbound.md: Breakup Email (Day 35).

Core structure: 2-3 sentences max. Acknowledge the silence without pressure. Leave the door open with a future trigger offer. Never apologize for reaching out.

Example: "I've sent a few notes about [specific angle from T1] — I'll leave it there. If [specific outcome, e.g., cutting regression time] becomes a priority down the road, happy to pick this back up. [Name], Testsigma"

Rules: No em dashes. No "circling back." No "last attempt" framing. CTA = none (or future-trigger offer only). Length: 40-60 words max.
