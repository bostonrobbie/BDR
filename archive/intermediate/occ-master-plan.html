Key Takeaways for Our System

1. Quality > Volume

BDRs making 80+ calls/day with poor targeting underperform those making 50 calls with better research. Our system should optimize for research quality, not activity count. Top performers hit 14.5% meaningful conversation rates through better targeting.

2. Multi-Channel is Non-Negotiable

287% higher engagement from multi-channel vs single-channel. 90% of BDRs now multithread (up from 83%). Our 6-touch multi-channel sequence is aligned with industry best practice. Average 21 attempts per contact across channels.

3. Opportunity > Meeting

Industry shifting from "meetings booked" to "opportunities created" as primary KPI. 74% of BDRs hand off as Opportunity (not Lead). Meeting show rate (50-90%) is a critical gap. Our system should track the full chain: touch → reply → meeting → show → opportunity.

4. Time is the Scarcest Resource

SDRs spend just 2 hours/day actually selling. The rest is admin and research. Our agent system should maximize that 2 hours by doing all research and drafting in advance. Rob should spend his time sending and having conversations, not researching.

Agent Swarm Architecture

Coordinator + Worker pattern. A Supervisor Agent orchestrates 11 specialized worker agents. Each has a defined role, input/output contract, and safety constraints.


Supervisor (Coordinator)
├── Research Cluster
│   ├── Prospector Agent        (finds + qualifies prospects from Sales Nav)
│   ├── Research Agent          (builds person + company briefs with citations)
│   └── Signal Scanner Agent    (detects intent, job postings, funding, leadership)
├── Draft Cluster
│   ├── Message Writer Agent    (writes all touches in Rob's voice)
│   ├── Sequencer Agent         (schedules follow-ups, optimizes timing)
│   ├── Call Prep Agent         (builds call cards, discovery questions)
│   ├── Reply Triage Agent      (classifies replies, recommends next step)
│   └── Quality Gate Agent      (10 QC checks, blocks bad output)
└── Analysis Cluster
    ├── Pre-Brief Agent         (5-line batch summary from historical data)
    ├── Insights Agent          (weekly analytics and recommendations)
    └── Experiment Analyst Agent (A/B test evaluation and conclusions)

Full agent specifications, communication protocol, and safety constraints documented in AGENT_SWARM.md

Controls and Safety

ControlWhat It Does
Manual Send OnlyNo "send" button in the system. Only "Copy" buttons. Rob pastes and sends manually in Sales Nav and Gmail.
Approval WorkflowDrafts start as "draft" status. Rob reviews and marks "ready" before using. Can edit any field.
Quality Gate10 automated checks on every message. Failed messages are blocked until fixed. QC flags visible in UI.
Confidence ScoreEach draft shows what evidence was used (research data, proof point, sources). No black box.
No HallucinationCustomer names must exist in proof points table. Stats must match known data. Placeholder detection blocks "[Name]" style text.
Voice EnforcementNo em dashes, word count within range, one question max, opener variety check, no feature dumping.
Full Audit TrailEvery change to every record is logged with who changed it (user or agent), when, and what the old/new values were.

A/B Experimentsexperiments + batch_prospectsControlled tests of one variable at a time. Lock in winners, test new hypotheses.
