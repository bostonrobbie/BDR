3. Cold Outbound Pipeline (Batch Workflow)

Phase 1: Pre-Brief (5 min)

Before building any new batch, Claude reads all previous prospect-outreach-*.html files and generates a 5-line "What's Working" summary covering: best persona, best proof point, best vertical, best pattern, and one thing to stop doing.

Phase 2: Prospect Sourcing (30-45 min)

Source 25 prospects from Sales Navigator saved searches. Apply the Prospect Mix Ratio:

PersonaTarget CountReply Rate
QA Manager / Lead10-1226.8%
QA Director / Head4-626.0%
Architect / Sr IC3-539.3%

VP Eng / CTOMax 211.9% / 9.1%

Check for no prior interaction ("Messaged:" or "Viewed:" indicators). US-based only. Software QA only (not pharma manufacturing). Max 2 from same company per batch.

Phase 3: Research & Enrichment (60-90 min)

For each prospect, gather data from all 3 sources:

SourceToolWhat to Extract
LinkedIn ProfileSales Navigator (browser)QA scope, team signals, tech stack clues, pain signals, career trajectory
Apollo EnrichmentApollo MCP APIVerified email, phone, title, tech stack, company size, industry, funding
Company ExternalWeb search + browsingProduct pages, eng blogs, job postings, press releases for QA-relevant signals

Tag each research bullet with which message element it feeds: opener, context, proof match, or close. If it doesn't feed any element, don't log it.

Phase 4: Message Drafting (60-90 min)

Write all touches per prospect following C2 style rules. Each prospect gets:

TouchChannelDayLengthContent
1InMail080-120 wordsFull C2 message (opener, context, proof, close)
2Cold Call33-line snippetPersonalized call cheat sheet
3InMail Follow-up540-70 wordsNew angle/proof point, softer close
4Cold Call #283-line snippetDifferent angle from Call #1
5Email1080-120 wordsReferences LinkedIn outreach, new proof point
6InMail Break-up1530-50 wordsRespectful close-out, no pitch

Phase 5: QA Gate (15 min)

Run all 14 QA checks on every message before presenting the batch to Rob. Every message must score MQS ≥9/12. Key checks include: Hard Constraint scan, word count (80-120), question count (exactly 2), close validation ("what day works" + proof point tie-in), hyphen audit (max 1), paragraph spacing (4+ breaks), and research depth (all 3 sources).

Phase 6: Build HTML Deliverable (15 min)

Generate the batch tracker HTML with prospect cards, copy-paste messages, call snippets, objection maps, priority scores, A/B groups, and personalization scores. Save to /mnt/Work/prospect-outreach-[batch#]-[date].html.

Phase 7: Apollo Contact Creation & Enrollment (10 min)

After Rob approves the batch:

  1. Create Apollo contacts for each prospect (with dedup check)
  2. Enroll contacts in "Q1 QA Outreach - US" sequence (ID: 699f4089628b940011da7fb7)
  3. This creates manual tasks in Apollo for LinkedIn sends and email sends

Phase 8: Send Session (60-90 min for 25 prospects)

Follow the LinkedIn Sales Navigator Live Send SOP. One prospect at a time. Pre-flight checks, compose message, present for approval, Rob clicks APPROVE SEND, post-send logging. Target: 2-3 minutes per prospect, with breaks every 10 sends.

5. Warm / Inbound Lead SOP

When This SOP Applies

Use this workflow when a lead comes in through any inbound channel: website demo request, content download, event registration, referral from existing contact, or Buyer Intent signal from Sales Navigator. These leads are warmer and need faster, more tailored handling than cold outbound.

Inbound Lead Sources

SourceDetection MethodResponse SLA
Buyer Intent (Sales Nav)Sales Navigator alerts or manual checkSame day
Referral from reply"Talk to [name]" in reply tagSame day
Re-engagement triggerNew job posting, funding, leadership changeWithin 48 hours
Website demo requestRob notifies ClaudeWithin 2 hours
Event/webinar attendeeRob shares attendee listWithin 24 hours

Warm Lead Workflow

1
Identify & Enrich — Apollo person + org enrichment. Verify email and phone. Check for existing Apollo contact (dedup). Check for prior outreach in batch trackers.
↓
2
Quick Research — LinkedIn profile scan + one external source (prioritize speed over depth). Identify the specific trigger that makes them warm.
↓
3
Draft Personalized Response — Reference the warm trigger directly. Shorter than cold outreach (50-80 words). More direct ask since they've shown interest. Use "what day works this week" (tighter timeframe than cold).
↓
4
Rob Reviews & Sends — Present draft for approval. Rob sends via the appropriate channel (email for demo requests, InMail for Buyer Intent, email for referrals).
↓
5
Fast Follow-Up — If no reply in 48 hours, send follow-up via alternate channel. If referral, CC the referring person. Calendar reminder for Day 2 and Day 5 follow-ups.

9. Credit Budget Management

Apollo Credits

ResourceAvailablePer Batch (25)Batches Remaining
Lead Credits~6,879~40-45~153-172
Direct Dial Credits600~10 (for call touches)~60
AI Credits50MMinimalEssentially unlimited

At 1 batch/week, lead credits last ~3 years. No urgency to conserve, but no reason to waste either.

LinkedIn InMail Credits

Check InMail credit balance before each send session. Sales Navigator typically provides 50 InMail credits/month. At 25 prospects/batch with 2 InMails each (Touch 1 + Touch 3), that's 50 InMails per batch, one full batch per month. Touch 6 break-ups add ~25 more if all prospects reach that stage.

If credits run low, prioritize Hot (5) and Warm (4) priority prospects. Defer lower-priority prospects to the next credit cycle.
