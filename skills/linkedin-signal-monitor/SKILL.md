## Purpose

Passive monitoring of LinkedIn engagement signals without taking any visible action. Surfaces profile views, InMail read receipts, connection request acceptances, and prospect activity so Rob can act with intent.

**Key principle:** This is observation-only. No likes, comments, reactions, messages, or connection requests through this skill. Rob reviews signals and decides action.

---

## Safety Rules (ENFORCE BEFORE EVERY ACTION)

1. **Read-only.** No likes, comments, reactions, posts, messages, or connection requests through this skill.
2. **Always use blue/work Chrome profile (Testsigma).** If red/personal profile is active, STOP and alert Rob immediately.
3. **Never take visible action.** No coworker can see activity generated by this skill.
4. **Stop on navigation error.** If LinkedIn blocks, redirects, or errors occur, stop and report — do not retry actions that could trigger notifications.
5. **No InMail sends.** Signal monitoring is read-only. All InMail strategy is Rob's decision.

## Execution Phases

### Phase 1: Verify Browser Profile

Before any LinkedIn navigation, establish safe browser state:

1. Use Chrome MCP `read_page` to capture browser state
2. Verify current profile shows: **Testsigma** / **Robert Gorham** (work account)
3. **If personal profile detected:** STOP. Alert Rob:
   ```
   ⚠️ Work Chrome profile not active.
   Please switch to the blue Testsigma profile before running LinkedIn monitoring.
   ```
4. If verified as work profile, proceed to Phase 2

---

### Phase 2: Check Who Viewed Your Profile

Navigate to LinkedIn profile view notifications:

1. Go to **linkedin.com/me** (Rob's LinkedIn profile page)
2. Click **Notifications** bell icon (top right)
3. Filter by **"viewed your profile"** if filter exists; otherwise scroll through recent notifications
4. Capture each viewer:
   - Name
   - Job title
   - Company
   - Date viewed (relative, e.g., "2 days ago")
   - URL/profile link (if present)

5. **Cross-reference Phase 2A: MASTER_SENT_LIST.csv**
   - Was this person already contacted in any sequence?
   - If yes: note the contact ID, sequence name, and touch number

6. **Cross-reference Phase 2B: warm-leads.md**
   - Is this an active warm lead (P0, P1, P2, P3)?
   - If yes: note current stage and last interaction date

7. **Classify each viewer into one of four buckets:**

   | Signal | Meaning | Action Hint |
   |--------|---------|-------------|
   | 🔥 **HOT** | Active warm lead (P0/P1) who just viewed profile | Reply opportunity — maybe they're ready to engage |
   | 👀 **PROSPECT** | In MASTER_SENT_LIST but not warm (P2/P3 or past contact) | They saw something, signal of interest |
   | 🆕 **NEW** | Not in any list — unknown prospect | Research trigger, potential inbound |
   | 🔄 **RE-ENGAGE** | Was in Do Not Contact list but viewed profile | Flag for Rob's decision — may want to re-open conversation |

---

### Phase 3: Check InMail Engagement

Navigate to LinkedIn Sales Navigator and check InMail read receipts:

1. Go to **linkedin.com/sales/ssi** (Sales Navigator) if not already there
2. Navigate to **Sent InMails** (or similar view in Sales Navigator)
3. Look for InMails marked as **"Read"** in the last 48 hours
4. For each read InMail, capture:
   - Recipient name
   - Company
   - Date sent
   - Date read (if visible)
   - Subject line (if visible)

5. **Cross-reference with MASTER_SENT_LIST.csv:**
   - Which sequence is this contact in?
   - Which touch is this InMail? (T1, T2, T3, etc.)
   - Days since send (calculate)

6. **Classify as:**
   - **Read + No Reply Yet:** Positive signal — they engaged but haven't responded
   - **Read + Reply Pending:** Check memory/session/messages.md and warm-leads.md for any reply draft or next action
   - **Read + Replied:** Mark as conversion signal for learning loop

---

### Phase 4: Check Connection Activity

Navigate to LinkedIn My Network and check pending/recent connections:

1. Go to **linkedin.com/mynetwork** (My Network)
2. Click **Pending** tab to see recent connection acceptances
3. For each accepted connection since last check, capture:
   - Name
   - Job title
   - Company
   - Date accepted

4. **Cross-reference with MASTER_SENT_LIST.csv:**
   - Were they also in an email or InMail sequence?
   - If yes: note the sequence and current touch

5. **Flag for Rob:**
   - Connection accepted = warm signal
   - Consider follow-up strategy (InMail, email, or message?)

---

### Phase 5: Compile Signal Report

Build a unified signal summary and output to two locations.

**Format:**

```
LinkedIn Signals — [YYYY-MM-DD] [HH:MM]

PROFILE VIEWS (last 24h):
- [Name] | [Title] | [Company] — [HOT/PROSPECT/NEW/RE-ENGAGE] | [Brief action hint]
  [Additional context if relevant, e.g., "2 days after T1 InMail"]

INMAIL READS (last 48h):
- [Name] | [Company] — Read [X] days after send | Sequence: [Seq Name] Touch [T#]
  Status: [Read + No Reply / Read + Pending Reply / Awaiting response]

NEW CONNECTIONS:
- [Name] | [Title] | [Company] — Connected | In sequence: [Yes/No]

SUMMARY:
[N] profile views, [N] InMail reads, [N] new connections.
Top priority: [Name/action recommendation].
```

**Output Destinations:**

1. **Append to `memory/session/messages.md`** under today's date (use date header if exists, create if not)
2. **Append to `memory/linkedin-signals.md`** as new entry at top (reverse chronological)

---

### Phase 6: Feed Morning Briefing

The `morning-briefing` skill checks `memory/linkedin-signals.md` for the most recent signals. Ensure output format is consistent:

- Use the exact format from Phase 5
- Include classification tags (🔥, 👀, 🆕, 🔄) so morning-briefing can parse and highlight
- Always include "SUMMARY" line so morning-briefing can extract top priority

## Learning Loop Integration

Follow `skills/_shared/learning-loop.md`:

1. **Log every run** in `run-log.md` with:
   - Timestamp
   - Number of signals found (profile views, InMail reads, connections)
   - Any errors or unusual conditions

2. **Track signal-to-action conversion** in `learned-patterns.md`:
   - Which signal types most often convert to replies/meetings?
   - Profile view → reply rate?
   - InMail read → reply rate?
   - Connection acceptance → follow-up rate?
   - Which companies/titles generate the most signals?

3. **Every 5th run (or weekly), review** `learned-patterns.md`:
   - Are hot signals actually converting?
   - Should classification thresholds change?
   - Which signal sources are most actionable?
   - Any patterns that should be fed back to other skills?

## Technical Notes

**Chrome Profile Verification:**
Use Chrome MCP `read_page` to inspect the current page for Testsigma/Robert Gorham indicators. If personal profile shows, return early with alert.

**LinkedIn Navigation:**
- Use Chrome MCP `navigate` to move between pages
- Use `find` to locate buttons and links (e.g., notifications bell, "viewed your profile" filter)
- Use `computer` for clicks if coordinates are needed

**Data Integrity:**
- All cross-references with MASTER_SENT_LIST.csv should use exact name/company match
- If person is not in MASTER_SENT_LIST but is in warm-leads.md, mark as warm signal
- If person appears in both lists with different stages, use warm-leads.md as source of truth (more recent)

**Rate Limiting:**
LinkedIn may rate-limit or show partial data. If navigation stalls or 429 errors occur, stop and alert Rob. Do not retry excessively.

## Error Handling

| Scenario | Action |
|----------|--------|
| Personal Chrome profile detected | STOP. Alert Rob. Exit. Do not navigate. |
| LinkedIn navigation error (404, timeout, 429) | STOP. Report error. Do not retry. |
| MASTER_SENT_LIST.csv not found | Log warning. Continue with partial cross-reference. |
| warm-leads.md not found | Log warning. Continue without warm-lead flagging. |
| No signals found | Log in run-log. Write "No signals found in [24h window]." to memory/linkedin-signals.md. |

## When NOT to Run

- If Rob's LinkedIn is restricted or locked by LinkedIn for security reasons
- If a LinkedIn interface redesign breaks navigation paths (stop and alert Rob; update paths in next iteration)
- If Rob explicitly says "skip LinkedIn checks" (e.g., during outreach pause or account freeze)
